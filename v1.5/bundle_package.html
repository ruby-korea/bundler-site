bundle package

젬을 ./vendor/cache에 고정하고 캐시함 
$ bundle package [--no-prune] [--all]

옵션:

--no-prune: 오래된 젬을 캐시에서 제거하지 않습니다.

--all: :git, :path, .gem 의존성을 패키징. 한 번 사용되면 --all 옵션은 기억됩니다.

package 명령어는 번들 안의 젬을 위한 .gem 파일을 ./vendor/cache에 복사합니다. 그 후 bundle install을 실행하면 번들러는 rubygems.org보다 높은 우선순위로 캐시 안의 젬을 사용합니다.

추가적으로 버전 관리 저장소에 캐시 디렉터리를 체크인하면 다른 사람들이 체크아웃할 때 추가적인 젬의 다운로드 없이 설치하게 할 수 있습니다.

RubyGems의 젬을 ./vendor/cache에 고정하고 캐시함 
$ bundle package

기본 값으로 bundle package 후에 bundle install을 실행하면 번들러는 여전히 rubygems.org에 접속해 vendor/cache안의 젬 중 플랫폼 지정 젬이 있는지 확인합니다.

이 행동은 bundle install --local을 실행함으로써 회피할 수 있습니다. 이 명령은 캐시된 모든 젬에 대해 정확한 플랫폼 버전이 필요하니 주의하세요. 이 목표를 달성하기 위한 가장 쉬운 방법은 동일한 기기에서 bundle package를 실행하고 벤더의 젬을 확인하는 것입니다.

젬을 ./vendor/cache에 고정, 캐시하고 오래된 젬을 캐시에서 제거하지 않음. 
$ bundle package --no-prune
:git, :path, .gem 의존성을 포함해 젬을 ./vendor/cache에 고정하고 캐시함. 
$ bundle package --all

한 번 사용되면 --all 옵션은 기억됩니다.

이 옵션은 번들러 2.0에서 기본 값이 됩니다.
