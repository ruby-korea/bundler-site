bundle install

애플리케이션이 Gemfile 안에 있는 의존성을 만족하는지 확인합니다. 
$ bundle install [--binstubs=PATH] [--clean] [--deployment] [--frozen]
                 [--full-index] [--gemfile=FILE] [--local] [--no-cache]
                 [--no-prune] [--path=PATH] [--quiet] [--shebang=STRING]
                 [--standalone=ARRAY] [--system] [--without=GROUP GROUP]
                 [--trust-policy=SECURITYLEVEL]

옵션:

--binstubs: ./bin 폴더에 실행 파일의 스텁을 생성합니다.

--clean: 설치 후 자동으로 bundle clean을 실행합니다.

--deployment: 배포 환경에 최적화된 기본 값으로 설치합니다.

--frozen: 이번 설치 이후 Gemfile.lock을 업데이트 할 수 없게 합니다.

--full-index: API 엔드포인트 대신 rubygems의 모던 인덱스를 사용합니다.

--gemfile: Gemfile 대신 지정한 젬파일을 사용합니다.

--jobs: 병렬 워커를 사용해 설치합니다.

--local: 원격에서 gem을 가져오지 않는 대신 캐시된 gem을 사용합니다.

--no-cache: 갱신에 캐시된 gem을 사용하지 않습니다.

--no-prune: 캐시에서 오래된 gem을 제거하지 않습니다.

--path: 시스템 기본 값($BUNDLE_PATH나 $GEM_HOME) 대신 다른 경로를 지정합니다. 번들러는 이 기기에서 이 값을 기억할 것입니다.

--quiet: 경고와 에러만 출력합니다.

--retry: 실패한 네트워크, git 요청을 재시도합니다.

--shebang: 시스템 기본 값(보통 'ruby') 대신 다른 쉬뱅(#!) 실행 이름을 지정합니다.

--standalone: 번들러 런타임 없이 실행되는 번들을 만듭니다.

--system: 시스템($BUNDLE_PATH 나 $GEM_HOME)에 설치합니다. 이전에 이 애플리케이션의 번들이 어디에 설치되었든지 관계없습니다.

--trust-policy: 인증된 gem을 다루기 위한 보안 수준을 설정합니다. 값으로 `LowSecurity`, `MediumSecurity`, `HighSecurity`를 사용할 수 있습니다.

--without: 지정한 그룹의 gem을 제외합니다.

gem은 기본 시스템에 설치될 것입니다. 시스템 gem이 루트 권한의 장소(맥 OSX처럼)에 설치 된다면 번들은 설치를 위해 관리자 비밀번호를 물어볼 것입니다.

gem을 설치할 동안, 번들러는 vendor/cache와 시스템 gem을 체크합니다. gem이 캐시되거나 설치되지 않았다면, 번들러는 Gemfile에서 선언한 소스에서 설치할 것입니다.

--system 옵션은 기본 값입니다. 아래에 설명된 대로 --path 옵션을 사용한 이후에 기본 값으로 되돌리고 싶을 때 넘기세요.

이미 시스템 gem에 설치되어 있어도 시스템 gem의 저장소가 아닌 곳에 의존성을 설치합니다. 이 경우에는, vendor/bundle에 설치합니다. 
$ bundle install --path vendor/bundle
또한 bundle 명령어, Bundler.setup, Bundler.require를 위해 이 위치를 기억해둡니다.

명시적으로 제외한 그룹 외의 모든 의존성을 설치합니다. 
$ bundle install --without development test
자세한 내용은: 그룹
프로덕션 서버에 모든 의존성을 설치합니다. 개발 기기에서 이 플래그를 사용하지 마세요. 
$ bundle install --deployment

--deployment 플래그는 여러 가지의 배포친화적인 규칙을 활성화시킵니다.

  • 모든 gem을 vendor/bundle에 격리합니다.
  • 최신 Gemfile.lock을 필요로 합니다.
  • bundle package가 실행되었다면, rubygems.org에서 가져오지 않고, vendor/cache 안의 gem만 사용합니다.
자세한 내용은: 배포하기
지정한 수의 워커를 사용하여 동시에 여러 개의 gem을 설치합니다. 
$ bundle install --jobs 4
실패한 네트워크, git 요청을 재시도합니다. 
$ bundle install --retry 3
